# 三資產 Mean-Variance Efficient Frontier

本分析建立聯發科、第一金與黃金的三資產均異效率前緣。黃金原始報價為美元，因此先用 USD/TWD 匯率轉成台幣計價，再和兩檔台股一起計算日報酬、共變異數矩陣、蒙地卡羅投組與最佳化配置。

## 結論

若以風險調整後報酬作為主要判準，建議採用 Maximum Sharpe 配置：

| 資產 | 配置比例 |
|---|---:|
| 聯發科 | 10.33% |
| 第一金 | 37.14% |
| 黃金（台幣計價） | 52.53% |

此配置的年化報酬為 `18.64%`，年化波動為 `12.51%`，Sharpe ratio 為 `1.490`。在 2021-2025 樣本期間中，聯發科提供最高年化報酬，但波動也最高；黃金與兩檔台股的相關性很低，因此 Maximum Sharpe 配置以黃金為主，搭配第一金與少量聯發科來平衡報酬與風險。

## 三種最佳化結果

| 投組 | 年化報酬 | 年化波動 | Sharpe | 聯發科 | 第一金 | 黃金 |
|---|---:|---:|---:|---:|---:|---:|
| 最大報酬 | 26.18% | 37.84% | 0.692 | 100.00% | 0.00% | 0.00% |
| 最大 Sharpe | 18.64% | 12.51% | 1.490 | 10.33% | 37.14% | 52.53% |
| 最小變異 | 17.15% | 12.00% | 1.429 | 3.22% | 54.00% | 42.78% |

解讀：

- 最大報酬投組全部配置到聯發科，因為聯發科在樣本期間的年化平均報酬最高。
- 最大 Sharpe 投組以黃金為主，並加入第一金與少量聯發科，因為黃金與台股相關性低，能提升分散效果。
- 最小變異投組以第一金與黃金為主，少量配置聯發科，用來降低整體波動。

## 相關係數

|  | 聯發科 | 第一金 | 黃金 |
|---|---:|---:|---:|
| 聯發科 | 1.000 | 0.284 | 0.029 |
| 第一金 | 0.284 | 1.000 | 0.084 |
| 黃金 | 0.029 | 0.084 | 1.000 |

黃金與聯發科的相關係數只有 `0.029`，黃金與第一金的相關係數也只有 `0.084`，代表黃金在樣本期間與台股日報酬連動性很低。這是黃金在 Maximum Sharpe 投組中占比高的主要原因之一：它能在保留報酬的同時提供分散效果。

## 模型設定

- 樣本期間：`2021-01-01` 到 `2025-12-31`
- 實際價格資料區間：`2021-01-04` 到 `2025-12-31`，因為 `2021-01-01` 非交易日
- 年化交易日：`252`
- 無風險利率：`0%`
- 蒙地卡羅模擬次數：`50,000`
- 隨機種子：`42`
- 效率前緣目標報酬點數：`120`
- 權重限制：每項資產介於 `0%` 到 `100%`
- 投組限制：權重加總等於 `100%`
- 放空限制：不允許放空

## 資料來源與計價

| 資產 | 代號 | 計價處理 |
|---|---|---|
| 聯發科 | `2454.TW` | 台幣，使用調整後收盤價 |
| 第一金 | `2892.TW` | 台幣，使用調整後收盤價 |
| 黃金 | `GC=F` | 美元報價，轉成台幣 |
| 匯率 | `TWD=X` | USD/TWD，也就是 1 美元兌台幣 |

黃金台幣價格：

```text
Gold (TWD) = GC=F close price in USD * USD/TWD
```

主資料來源為 Yahoo chart API。程式也保留公開資料備援：台股使用 TWSE，匯率使用中央銀行統計資料庫，黃金使用 FreeGoldAPI。

## 執行方式

在 `efficient-frontier` 資料夾執行：

```powershell
python mean_variance_frontier.py
```

執行後會產生或更新 `outputs/` 內的資料與圖檔。
